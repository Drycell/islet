# Branch Policy
* According to condition of proceeding the project and scale of team,
we should choose a proper branch model and define the branches to use and
the workflow between them.

	* This project basically follows level 1 (Single branch) branch model.

		- Only one branch (name : main) to manage code
			both for development and deployment.

* Merging to master branch must be allowed only when the pull request achieves
at least one approval as its review.
